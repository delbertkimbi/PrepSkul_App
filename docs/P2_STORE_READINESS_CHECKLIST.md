## App Metadata

- [ ] App name, short description, and full description finalized.
- [ ] Category and tags aligned with educational use case.
- [ ] Contact email and support URL are valid.

## Policy & Legal

- [ ] Privacy Policy URL is live and accurate.
- [ ] Terms of Service URL is live.
- [ ] Data safety form matches actual app behavior.
- [ ] Permissions declarations have clear purpose text.

## Access & Reviewer Instructions

- [ ] Test account credentials prepared for review.
- [ ] App access instructions documented (especially gated features).
- [ ] SkulMate generation limitations explained for reviewers if needed.

## Media Assets

- [ ] App icon and feature graphic are final.
- [ ] Phone screenshots updated for latest UI.
- [ ] Screenshot set includes SkulMate upload + gameplay + results.
- [ ] Optional promo video links validated.

## Build & Release

- [ ] Release AAB built from tagged commit.
- [ ] Version code/version name incremented correctly.
- [ ] Signing config and keystore are validated.
- [ ] Proguard/R8 mapping upload path prepared (if used).

## Quality Gate Before Submit

- [ ] No blocker regressions in release smoke checklist.
- [ ] Crash monitoring enabled and alert routes tested.
- [ ] Rollback strategy documented.

## Submission

- [ ] Release notes prepared and user-facing.
- [ ] Staged rollout percentage defined.
- [ ] Post-release monitoring owner assigned.
